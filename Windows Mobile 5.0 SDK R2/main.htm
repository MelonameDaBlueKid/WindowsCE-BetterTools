Windows Mobile Version 5.0 SDK

The Windows Mobile Version 5.0 SDK package contains documentation, sample code, header and library files, and tools that help you support the new features of Windows Mobile Version 5.0. This document contains important information about this package. For general information about writing software for Windows Mobile, please see the Windows Mobile - Developer Home.

Requirements

Before you can install the SDK, you must previously have installed Visual Studio 2005. Please read and take note of all requirements for that package, which are detailed at length on the appropriate download pages.

After you have installed Visual Studio 2005, study the readme file (readme.htm) located in the Visual Studio 8, Program Files folder. It contains important late-breaking information about Smart Device development in Visual Studio 2005.

Installation

Install the SDK package by double-clicking on the compressed archive. Please disable any virus checking software prior to installation.

Development Notes

Warning Message on Initial Application Deployment

Bootstrap file for corecon on x86 is signed with a certificate that may not be provisioned to your device;this causes the unsigned/untrusted app warning message to be shown on pocket pc emulator images. If this is dismissed once it won't be displayed again until the emulator is rebooted cleanly.

"New Project" Code Wizards

Windows Mobile v.5.0 code wizards are improperly named "Smartphone Magneto" or "Pocket PC Magneto". These names are placeholders generated by Visual Studio for Devices during development and do not imply release date or product name.

No ActiveSync over Ethernet with Windows Mobile Version 5.0 devices

The "RemoteSync" feature of ActiveSync has been disabled in this release, as Microsoft investigates the underlying security of the model.

New Pocket PC Security Model

Pocket PC now supports a new security model, similar to that of Smartphone. Documentation on this will be posted, when available, off of the MSDN Windows Mobile Developer Center, located at: http://msdn.microsoft.com/mobility/windowsmobile/default.aspx.

GAPI applications that are NOT full screen will not function in the Device Emulator

This is a known limitation that will be resolved in a future release of Windows Mobile
