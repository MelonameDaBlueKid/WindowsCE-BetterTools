Windows Mobile 6.5.3 Localized Emulator Images

This package includes localized Windows Mobile 6.5.3 emulator images that can be used with Visual Studio to test applications on different localizations or simply as standalone emulators without Visual Studio:

1.  Integration with Visual Studio

2.  Standalone Usage

3.  Cellular Emulator

4.  Feedback

1. Integration with Visual Studio

The Windows Mobile 6.5.3 Localized Emulator Images package adds localized emulator images to Visual Studio 2005 and Visual Studio 2008 that let you test applications for Windows Mobile 6.5.3 on different locales. This document contains important information about this package. For general information about writing software for Windows Mobile, please see the Windows Mobile Developer Center.

The new emulator images are automatically registered with Visual Studio during installation, and are available as options when deploying applications form both native and managed projects.

2. Cradling

The Windows Mobile Emulator Images can be connected to the desktop (cradling) through ActiveSync 4.5 or Windows Mobile Developer Center on Windows Vista and Windows 7:

1.  Make sure ActiveSync 4.5 or Windows Mobile Developer Center (on Windows Vista or Windows 7) is installed.

2.  Launch the Emulator Image following the steps on item 2 above.

3.  Run Device Emulator Manager:

o   Click on Start / Run;

o   Type "C:\Program Files\Microsoft Device Emulator\1.0\dvcemumanager.exe" and then click on OK;

o   Under Others, right-click the running Emulator Image, and then click Cradle.

3. Cellular Emulator

The Cellular Emulator introduced in the Windows Mobile 6 SDK Refresh is not being supplied with this DTK release.
