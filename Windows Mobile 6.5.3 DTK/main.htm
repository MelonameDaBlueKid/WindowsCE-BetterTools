Windows Mobile 6.5.3 DTK

The Windows Mobile 6.5.3 DTK adds documentation, sample code, header and library files, emulator images and tools to Visual Studio that let you build applications for Windows Mobile 6.5 and Windows Mobile 6.5.3. This document contains important information about this package. For general information about writing software for Windows Mobile, please see the Windows Mobile Developer Center.

What’s New in the Windows Mobile 6.5.3 DTK?

Developer Experience Features Description

Windows Mobile 6.5

Windows Mobile 6.5 and Windows Mobile 6.5.3 contain significant enhancements from prior versions of Windows Mobile. Of special note are the Gesture APIs. These APIs and other aspects of Windows Mobile 6.5 are documented in the Windows Mobile 6.5 documentation. This documentation includes information about Windows Mobile 6.5.3 and is included with this release of the DTK. It can be accessed as a standalone collection through the Start Menu or by using "F1" inside Visual Studio (Visual Studio 2005 only). It is available from the Start menu in the Windows Mobile 6.5.3 DTK installation, where it is called Windows Mobile 6.5.3 DTK Help.

Relation to Prior Versions

This Windows Mobile 6.5.3 DTK should be installed after the Windows Mobile 6 SDK Refresh is installed to provide the base tools. The installation of this DTK will not overwrite anything from the SDK. The two versions co-exist side by side in Visual Studio and it is possible to write concurrently for both Windows Mobile 6 and Windows Mobile 6.5.

Documentation

Windows Mobile 6.5 documentation includes information about Windows Mobile 6.5.3 and is included with this release of the DTK. This documentation provides information about Windows 6.5 features and APIs. It can be accessed as a standalone collection through the Start Menu or by using "F1" inside Visual Studio (Visual Studio 2005 only). It is available from the Start menu in the Windows Mobile 6.5.3 DTK installation, where it is called Windows Mobile 6.5.3 DTK Help.

Windows Mobile Classic

There is no Windows Mobile 6.5 (or later) version of Windows Mobile Classic.

Visual Studio 2008

The Windows Mobile 6.5.3 DTK may be used with Visual Studio 2008 as well as with Visual Studio 2005.

A new code sample is provided that uses the Mobile Mapping Framework. The folder is ...\Samples\Common\CS\MMF.

There is now support for native control theming.

Device Emulator.exe, which is required to load images, is not included in the DTK. It installs with Visual Studio 2008 or can be downloaded from Microsoft Device Emulator 3.0, Standalone Release.

New Tools

xHopper – A revision of Hopper for Windows Mobile 6.5. Like Hopper, xHopper is a software test tool that simulates input stress on Windows Mobile devices. xHopper stresses all applications made available through the menu system by rapidly sending keystrokes and screen taps in a random fashion. Please see the topic xHopper Test Tool QuickStart Guide in the Windows Mobile 6.5 documentation that is included with this release of the DTK. It can be accessed as a standalone collection through the Start Menu or by using "F1" inside Visual Studio (Visual Studio 2005 only). It is available from the Start menu in the Windows Mobile 6.5.3 DTK installation, where it is called Windows Mobile 6.5.3 DTK Help.

What Was New in the Windows Mobile 6 SDK Refresh?

The following features were new in the Windows Mobile 6 SDK Refresh release which preceded this release, and still apply to this release.

Developer Experience Features Description

Windows Mobile Categories

With Windows Mobile 6, we are revising our SKU taxonomy and naming to better align our brand and products with the realities of today’s mobile device marketplace. The historical form-factor based distinction between Windows Mobile powered Smartphone and Windows Mobile powered Pocket PC Phone Edition is blurring dramatically. We want our taxonomies and terminology to evolve to better reflect the evolution of the mobile device industry.

The table below summarizes the changes:

Previous Categories

New Categories

Windows Mobile for Pocket PC

Windows Mobile Classic

Windows Mobile for Pocket PC Phone Edition

Windows Mobile Professional

Windows Mobile for Smartphone

Windows Mobile Standard

The Windows Mobile 6.5.3 DTK reflects these changes on the naming of "Project Templates" and "Emulator Images". However some folders still have names that reflect older categories such as “PocketPC” and “SmartPhone”.

.NET Compact Framework

The .NET Compact Framework is a hardware-independent environment for running programs on resource-constrained computing devices. It is a subset of the .NET Framework class library and also contains classes exclusively designed for it. It inherits the full .NET Framework architecture of the common language runtime and managed code execution.

The .NET Compact Framework 2.0 Service Pack 2 is now included in the ROM of Windows Mobile 6.5 devices.

Microsoft SQL Server™ 2005 Compact Edition

SQL Server Compact Edition is a lightweight relational database for use as a compact local database on a PC, Tablet PC, Windows Mobile device, or Windows CE device. Developers will be able to use the same database to develop occasionally-connected applications consistently across all Microsoft client platforms. The database can be used for local storage as well as for data synchronization with other editions of SQL Server 2005.

SQL Server Compact Edition is included in ROM of Windows Mobile 6 devices.

Device Emulator

Device Emulator 2.0 provides greatly improved performance at executing ARM instruction. It reduces cold-boot time and improves application execution speed. It provides high resolution input, the ability to change emulator options on-the-fly and has added driver support in the BSP for peripherals such as battery and headset/speakerphone with Windows Mobile 6.5 emulator images.

Windows Mobile Ink

Windows Mobile Ink is the ink service platform for Windows Mobile Classic and Professional. It provides full interoperability with ink data from Tablet PC through ISF stream and brings a rich inking experience, similar to Tablet PC, to Windows Mobile through high quality, curve-fitted ink. It provides APIs for Ink Collection, recognition, rendering, and Ink data management.

Sound APIs

A new set of APIs were introduced in Windows Mobile 6 to make it easier to manage and play sound. The new APIs support playing sounds in a variety of formats that Windows Media Player supports (MID, WMA, MP3, WAV, etc.) and provide the option to respect the phone’s profile when playing sounds.

These APIs are defined in <soundfile.h>.

Documentation

Documentation can be accessed as a standalone collection through the Start Menu or using "F1" inside Visual Studio.

New Tools

FakeGPS - Enables developers to receive data using the GPS APIs even if there is no GPS receiver on the device.

GPSSettings - Enables Windows Mobile Standard users to configure the GPS Intermediate driver. It works in the same way as the built-in GPS Settings applet on Windows Mobile Classic and Professional.

Hopper - A software test tool that simulates input stress on Windows Mobile powered devices. Hopper stresses all applications made available through the menu system by rapidly sending keystrokes and screen taps in a random fashion.

CabSignerTool - CabSignTool signs a CAB and all its executable content (.exe, .dll) with specified certificates.

FakeServer - A generic server library that enables users to write web services that run on the device locally.

Windows Mobile Test Framework - The Windows Mobile Test Framework contains a collection of classes, interfaces, and value types that allow your test code to access controls and resources on a Windows Mobile device.

Development Notes

Virtual Studio 2005 Standard is not supported for widget development.

Virtual Studio 2005 Standard is not supported for widget development. You must use the Professional or greater version of either Visual Studio 2005 or Visual Studio 2008.

F1 Help is not supported in Visual Studio 2008.

F1 does not provide access to the Windows Mobile 6.5.3 DTK Help in Visual Studio 2008. F1 does provide access in Visual Studio 2005.

Smart Device project templates for Windows Mobile 6.5.3 are not available in Visual Studio 2008.

Smart Device project templates for Windows Mobile 6.5.3 are available in Visual Studio 2005 but not in Visual Studio 2008.

Deploying managed applications to the emulator always causes the .Net Compact Framework to be deployed

When an application is deployed to the device or emulator, Visual Studio checks if .NET Compact Framework 2.0 is installed. If it is not installed Visual Studio deploys the framework and then the application. Although emulator images and Windows® phones that run Windows Mobile 6 have .NET Compact Framework 2.0 SP2 in the ROM, Visual Studio continues to check for the 2.0 release and every time tries to deploy it; however the installation fails because it is not possible to install an older version (2.0) on top of a newer version (2.0 SP2). Since the installation failed, the next time Visual Studio will try to deploy it again. To fix this behavior install the .NET Compact Framework 2.0 Service Pack 2.

The Windows Mobile 6.5.3 DTK installation rolls back

"Visual C++ Smart Device Programmability", which is an option of Visual Studio 2005 Setup, must be selected and installed before installing the Windows Mobile 6.5.3 DTK. If this feature is not installed, the Windows Mobile 6.5.3 DTK setup will roll back and not install properly.

Compiling Samples under Windows Vista

On Windows Vista, it may not be possible to compile the samples directly on the Windows Mobile 6.5.3 DTK Samples installation folder due to permissions. Copy the samples to a location where you have read and write permissions before opening them in Visual Studio.

Windows Server 2003 and DTK installation

Security polices can trigger at the Software Restrictions Polices level when the DTK loads into memory because of the size and number of files in the DTK. The installation fails with a message stating that the package was rejected by digital signature policy. A workaround is available here.
