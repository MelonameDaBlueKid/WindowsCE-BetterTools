Windows Mobile 6 SDK Refresh

The Windows Mobile 6 SDK Refresh adds documentation, sample code, header and library files, emulator images and tools to Visual Studio that let you build applications for Windows Mobile 6. This document contains important information about this package. For general information about writing software for Windows Mobile, please see the Windows Mobile Developer Center.

What’s New in the Windows Mobile 6 SDK Refresh?

Developer Experience Features Description
Windows Mobile 6

With Windows Mobile 6, we are revising our SKU taxonomy and naming to better align our brand and products with the realities of today’s mobile device marketplace. The historical form-factor based distinction between Windows Mobile powered Smartphone and Windows Mobile powered Pocket PC Phone Edition is blurring dramatically. We want our taxonomies and terminology to evolve to better reflect the evolution of the mobile device industry.

The table below summarizes the changes:

Previous Categories New Categories
Windows Mobile for Pocket PC Windows Mobile Classic
Windows Mobile for Pocket PC Phone Edition Windows Mobile Professional
Windows Mobile for Smartphone Windows Mobile Standard

The Windows Mobile 6 SDK Refresh reflects these changes on the naming of "Project Templates" and "Emulator Images".
Cellular Emulator Cellular Emulator is a software-based emulator to help developers and testers develop and test Windows Mobile platform applications. Cellular Emulator replaces the radio module, FakeRIL, in both development and test environments. The advantage of Cellular Emulator is that it provides not only voice but also data connectivity. Moreover, the Cellular Emulator is a powerful tool to test various applications under different wireless network conditions in GSM/GPRS and/or UMTS networks.

Cellular Emulator, or CellEmu, offers the following features:
  • CellEmu is a UI-driven application desktop application, making it easy to operate.
  • CellEmu device images incorporate an actual production radio driver.
  • CellEmu supports PPP data connection.

.NET Compact Framework

The .NET Compact Framework is a hardware-independent environment for running programs on resource-constrained computing devices - it is a subset of the .NET Framework class library and also contains classes exclusively designed for it. It inherits the full .NET Framework architecture of the common language runtime and managed code execution.

The .NET Compact Framework 2.0 Service Pack 2 is now included in the ROM of Windows Mobile 6 devices.
Microsoft® SQL Server ™ 2005 Compact Edition SQL Server Compact Edition is a lightweight relational database for use as a compact local database on a PC, Tablet PC, Windows Mobile powered, or Windows CE-based device. Developers will be able to use the same database to develop occasionally-connected applications consistently across all Microsoft client platforms. The database can be used for local storage as well as for data synchronization with other editions of SQL Server 2005.

SQL Server Compact Edition is included in ROM of Windows Mobile 6 devices.

Device Emulator Device Emulator 2.0 provides greatly improved performance at executing ARM instruction. It reduces cold-boot time and improves application execution speed. It provides high resolution input, the ability to change emulator options on-the-fly and has added driver support in the BSP for peripherals such as battery and headset/speakerphone with Windows Mobile 6 emulator images.
Windows Mobile Ink Windows Mobile Ink is the ink service platform for Windows Mobile Classic and Professional. It provides full interoperability with ink data from Tablet PC through ISF stream and brings a rich inking experience, similar to Tablet PC, to Windows Mobile through high quality, curve-fitted ink. It provides APIs for Ink Collection, recognition, rendering, and Ink data management.
Sound APIs A new set of APIs were introduced in Windows Mobile 6 to make it easier to manage and play sound. The new APIs support playing sounds in a variety of formats that Windows Media Player supports (MID, WMA, MP3, WAV etc.) and provide the option to respect the phone’s profile when playing sounds.

These APIs are defined in <soundfile.h>
Documentation Documentation can be accessed as a standalone collection through the Start Menu or using "F1" inside Visual Studio.
New Tools FakeGPS - Enables developers to receive data using the GPS APIs even if there is no GPS receiver on the device.

GPSSettings - Enables Windows Mobile Standard users to configure the GPS Intermediate driver. It works in the same way as the built-in GPS Settings applet on Windows Mobile Classic and Professional.

Hopper - A software test tool that simulates input stress on Windows Mobile powered devices. Hopper stresses all applications made available through the menu system by rapidly sending keystrokes and screen taps in a random fashion.

CabSignerTool - CabSignTool signs a CAB and all its executable content (.exe, .dll) with specified certificate(s).

FakeServer - A generic server library that enables users to write web services that run on the device locally.

Windows Mobile Test Framework - The Windows Mobile Test Framework contains a collection of classes, interfaces, and value types that allow your test code to access controls and resources on a Windows Mobile device.

Requirements

The following software is required. Please check each application's system requirements individually.

Windows Server 2003
  • Windows Server 2003 Service Pack 2.
  • Microsoft Visual Studio 2005, Standard Edition or above (Express Editions are not supported). SP1 recommended.
  • Microsoft .NET Compact Framework v2 SP2.
  • ActiveSync 4.5. If not currently installed, it is available from Microsoft Windows Mobile web site.
Windows XP
  • Microsoft Windows XP Service Pack 2.
  • Microsoft Visual Studio 2005, Standard Edition or above (Express Editions are not supported). SP1 recommended.
  • Microsoft .NET Compact Framework v2 SP2.
  • ActiveSync 4.5. If not currently installed, it is available from Microsoft Windows Mobile web site.
Windows Vista
  • Microsoft Windows Vista.
  • Microsoft Visual Studio 2005, Standard Edition or above (Express Editions are not supported). SP1 and SP1 Update for Windows Vista required.
  • Microsoft .NET Compact Framework v2 SP2.
  • For synchronizing data, the Windows Mobile Device Center is required. It is available from Microsoft Windows Mobile web site.

Important: "Visual C++ Smart Device Programmability", which is an option of Microsoft Visual Studio 2005 Setup, must be selected and installed during Visual Studio 2005 installation.

Please read and take note of all requirements for those packages, which are detailed on the appropriate download pages.

Development Notes

Windows Mobile Managed Assemblies

Installing Windows Mobile 5.0 SDK and Windows Mobile 6 SDK Refresh will cause duplication of the Windows Mobile managed assemblies in the “Add Reference…” dialog in Visual Studio 2005. Windows Mobile 6 managed assemblies are those installed in the "Windows Mobile 6 SDK" directory under "Program Files".

Toolbox is empty after installing Windows Mobile 6 SDK Refresh

If toolbox is empty after installing Windows Mobile 6 SDK Refresh, right click on the Toolbox window and click on "Reset Toolbox" - this will bring back all controls.

Deploying managed applications to the emulator always causes the .Net Compact Framework to be deployed

When an application is deployed to the device or emulator, Visual Studio checks if .NET Compact Framework 2.0 is installed - if it is not installed Visual Studio deploys the framework and then the application. Although Windows Mobile 6 powered devices and emulator images have .NET Compact Framework 2.0 SP2 in the ROM, Visual Studio continues to check for the 2.0 release and every time tries to deploy it; however the installation fails because it is not possible to install an older version (2.0) on top of a newer version (2.0 SP1) - as the installation failed, next time Visual Studio will try to deploy it again. To fix this behavior install the .NET Compact Framework 2.0 Service Pack 2.

Windows Mobile 6 SDK Refresh installation rolls back

"Visual C++ Smart Device Programmability", which is an option of Visual Studio 2005 Setup, must be selected and installed before installing Windows Mobile 6 SDK Refresh. If this feature is not installed, the Windows Mobile 6 SDK Refresh Setup will roll back and not install properly.

Compiling Samples under Windows Vista

On Windows Vista, it may not be possible to compile the samples directly on the Windows Mobile 6 SDK Samples installation folder due to permissions. Please, copy the samples to a location where you have read and write permissions before opening them in Visual Studio.

Microsoft Cellular Emulator does not support Windows XP Professional x64 Edition

The current version of the Cellular Emulator does not support Windows XP Professional x64 Edition. If you run the Cellular Emulator on Windows XP Professional x64 Edition, the OS returns an error message stating that there is not enough pairs of XPVCOM in the system.

Windows Server 2003 and SDK installation

Security polices can trigger at the Software Restrictions Polices level when the SDK loads into memory because of the size and number of files in the SDK. The installation fails with a message stating that the package was rejected by digital signature policy. A workaround is available here.
