// GSM Syntax Definition 
// =====================
//
// This file defines AT commands and responses understood by the GSM simulator.
// By editting this file, you can make the simulator model any hardware you
// like.
//
// These are the default command/responses.  You may edit a response at runtime
// using the GUI, in which case it will be saved to the registry and will always
// override the contents of this file.  If you specified a response here and 
// it's not being reflected in the simulation, open regedit and look at:
//
//      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Stinger\GSMS\GSMCommands
//



// Command/Response Patterns and Variables
// =======================================
//
// Each AT command and response is defined as a "pattern" or series of tokens. 
// Tokens may be constants or variables. 
//
// Patterns may be thought of like printf()/scanf() format strings.  One pattern
// is used to identify and validate an incoming AT command, and another pattern
// formats the outgoing response.
//
// Variables in a command pattern will be assigned values from the input
// string.  Variables in a response pattern will be replaced with that value.  
// In this way you can save the phone number dialed, or the contents of a
// "register" assignment, and then return it in response to a query.
//
// You may also pre-define variables, instead of assigning them from an AT 
// command.  A good example might be the "firmware" version the emulator should
// report.  See "GSM context-initialization" at the bottom of this file.
// 



// Format of a Pattern Token
// =========================
// 
// Pattern tokens have a type, and are declared as follows:
//   [Type:Range|Varname]      // the [,],:, and | chars are literal.
//
//  . Type is one of N,X,U,S,Y,V,H,O,INT, or TXT (see below).
//  . Range is x-y (lower and upper limits on an integer) or ANY (accepts any input).
//  . Varname is optional; if provided, the value of the token will be saved in
//    a variable named Varname.
//
//
// Token Types
// ===========
//
// Const Integer:       [N:x-y]         // input must fall between x and y
// 
// Const String:        [X:cccc]        // just a string.
// 
// Variable Integer:    [V:x-y|varname] // input value will be stored in varname
//
// Variable String:     [S:ANY|varname] // NOT USED?
// 
// Variable String:     [Y:ANY|varname] // input value will be stored in varname
//                                      // when varname used for output, the value
//                                      // will be returned in quotes: "var contents"
// 
// Variable Phone #:    [U:ANY|varname] // accepts any input 
//                                      // number will be stored in varname
//                                      // not sure why you'd use this instead
//                                      // of type S or Y.
// 
// 
// Response Pattern Header:
//      H:condition,repeat_count        // if(condition) then for(i=1..repeat_count)
//                                      //    send response
//                                      // Is repeat functionality implemented??
// 
// Command Operation:
//      O:varname                       // Perform arithmetical operation on a 
//                                      // variable.  For example, [O:varname++]
//                                      // will increment the variable.  
//                                      // Only ++ and -- currently supported?
//                                      // TODO: implement assignment: 
//                                      //      varname := value
// 
// GSMSim reserved command:
//      R:command                       // will invoke a function in the 
//                                      // simulator to do magic stuff.
//                                      // << investigate/document this >>
//
//


// Format of a Command/Response Mapping
// ====================================
//
// Command/response pairs are defined as follows:
//  
//  struct SGSMRespPattern
//  {
//  	LPCTSTR m_szKey;			// GSM comand key
//  	LPCTSTR m_szCmdPattern;		// GSM command pattern
//  	LPCTSTR m_szRespPattern;	// GSM response pattern
//  	LPCTSTR m_szSyntax;			// GSM response syntax
//  	LPCTSTR m_szDescription;	// GSM command description
//  	LPCTSTR	m_szErrCodes;		// posible error codes
//  };
//  
//
// Example:
//
//  +CCFC=
//  [X:+CCFC=][V:0-5|STNG_INT_CCFC_REASON],[V:0-4|STNG_INT_CCFC_MODE]
//  [X:OK]
//  OK
//  Call forwarding setting.
//  Default
//
//
// Let's take this entry one line at a time:
// 
// +CCFC=
//
//      The short form is used to match an input string to a command definition.
//      Also used as a registry key.
//
//
// [X:+CCFC=][V:0-5|STNG_INT_CCFC_REASON],[V:0-4|STNG_INT_CCFC_MODE]
//
//      The long form is used to parse/validate the input string:
//      [X:+CCFC=] 
//          - a constant string matching "+CCFC="
//      [V:0-5|STNG_INT_CCFC_REASON] 
//          - an integer between 0 and 5; saved to variable STNG_INT_CCFC_REASON
//      [V:0-4|STNG_INT_CCFC_MODE]
//          - an integer between 0 and 4; saved to variable STNG_INT_CCFC_MODE
//
//
// [X:OK]
//
//      The response string for this command will always be "OK" if the command
//      is successfully parsed.
//
//
// OK
//
//      The response syntax.  If a response is more than "OK" you should 
//      describe it here.  Informational purposes only.  See +CLCC for good
//      example.
//
//
// Call forwarding setting.
//      
//      A friendly description of this command.
//
// Default
//  
//      This command may return the Default error code.  The Default is hard-
//      coded in the simulator to "ERROR".  As near as I can tell this param 
//      is invariant/never used and should be removed from the simulator.
//
