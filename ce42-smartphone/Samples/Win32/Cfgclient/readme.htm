Sample Overview

CfgClient is a push client that receives a push message and then calls the public API DMProcessConfigXML on the message’s body, which should be UTF-8-encoded WAP provisioning XML (with no leading byte-order markers).  The resulting XML is then sent to the destination specified as the value of the X-MyCompany-SenderAddrType header (in the message’s headers section); this destination address must be either an http URL or a phone number (e.g., http://www.foo.com/bar.asp  or 14253334444).  Another header (X-MyCompany-SenderAddr) determines whether the destination address is a URL or a phone number (with values “http” or “wsp”, respectively).  Note that if the destination address is a URL, this client will POST the XML in URL-encoded format to that URL.  If the destination address is a phone number, this client will send the XML in WDP packets to port 9030 (with the source port being 9031).  The application id of this push client is “cfgclient”, and so push messages intended for this client should have “cfgclient” as the X-Wap-Application-Id header.

 

Source file listing and summary:

cfgclient.cpp – contains functions for processing the push message, formatting the reply and posting the reply.

main.cpp – contains the application entry point.

string.cpp – defines all constant string values.

utils.cpp – contains functions for extracting data from SMS header and copying strings.

fileclient.h – contains function prototypes and forward declaration for constant strings.

precomp.h – defines precompiled header.

Requirements and Dependencies

This sample requires the Pocket PC 2003 Phone Edition or Smartphone 2003 device and Microsoft eMbedded Visual C++ to be installed.

Building the Sample

Load the project in eMbedded Visual C++, pick the target platform and SDK, and build normally.

 

Running the Sample

Run the sample on the device once with “/register” as the command line argument.  After registering the sample receiving any SMS message with X-Wap-Application-Id set to “cfgclient” will result in the application running.
