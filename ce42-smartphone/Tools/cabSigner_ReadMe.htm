Tool Overview

A Win32 tool that cracks a CAB file, digitally signs all the relevant files inside the cab with a certificate, rebuilds the cab file, and signs the cab with the same certificate.  This is useful on Smartphone for ensuring that all of a package’s files meet the security requirements of a specific phone.

 

 

 

Usage

c:\>cabsigner.exe source destination [options] [-exesign options]

 

Signs a CAB and all its executable content (.exe, .dll) with the specified certificate(s).

 

CABSIGNER source destination [options]

(Uses the same certificate to sign both the CAB and its executable content.)

 

CABSIGNER source destination [options] –exesign [options]

(Uses the first certificate to sign the CAB and the second to sign its executable content.)

 

source: Name of unsigned input CAB file (.cab or .hme).

destination: Name of signed output CAB file.

options: Parameters to be used by signcode.exe. Use signcode.exe /? for a full list.

exesign: Specify this parameter and subsequent signcode.exe options to sign the executable contents with a different certificate.

 

 

Requirements

1. signcode.exe should be in a location pointed to by system path

2. makecab.exe should be in a location pointed to by system path

3. You should have the certificate files (.cer and .pvk file) handy. These files contain the keys used to sign your application.

 

 

 

Example

 

Suppose you want re-sign foo.cab. Here are the steps:

 

  1. Copy foo.cab to a new directory, for example, c:\Temp\Resign
  2. Copy the files cabsigner.exe, signcode.exe and makecab.exe to c:\Temp\Resign (if signcode and makecab are in the system path, you need not copy them)
  3. Copy the files cabpriv.cer and cabpriv.pvk to c:\Temp\Resign (these are the certificates you will use to sign the CAB.)
  4. Copy the files exepriv.cer and exepriv.pvk to c:\Temp\Resign (these are the certificates you will use to sign the executables in the CAB.)
  5. Execute the following cmd line:
    c:\Temp\Resign> cabsigner foo.cab foo_signed.cab -spc cabpriv.cer -v cabpriv.pvk -n "Signed using cabpriv.cer" -a sha1 -exesign -spc exepriv.cer -v exepriv.pvk -n "Signed using exepriv.cer" -a sha1
  6. If all goes well, this will result in a cab named foo_signed.cab in the c:\Temp\Resign directory

 

 

Signing a single EXE file:

~~~~~~~~~~~~~~~~~~~~~~~~~~

 

The following is a sample command line to sign an EXE file with signcode.exe

 

c:\Temp\Resign> signcode.exe -spc exepriv.cer -v exepriv.pvk -n "Signed using exepriv.cer" -a sha1 <filename>
