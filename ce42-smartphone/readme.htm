Requirements

1.     Processor: PC with a Pentium II-class processor, 450 MHz or higher (Pentium III-class, 600 MHz recommended)

2.     256MB RAM

3.     One of the following operating systems:

a.     Microsoft Windows 2000 Professional with Service Pack 3 installed

b.     Microsoft Windows 2000 Server with Service Pack 3 installed

c.     Microsoft Windows XP Professional with Service Pack 1 installed

4.     Display adapter and monitor capable of displaying a resolution of 1024x768 at 65536 colors (or higher)

5.     Microsoft eMbedded Visual C++ 4.0 installed with Service Pack 2 and/or Visual Studio .NET 2003

6.     You must be logged on using an account with local administrative rights on the computer onto which you are installing the SDK.

Installation Instructions

The following installation steps requires you to be logged onto to your computer using an account with Local Administrator privileges.

1. Close All Programs, Remote Development Tools, and Emulators Before Proceeding

You should close the following programs before proceeding with the Smartphone 2003 SDK installation:

§       eMbedded Visual Tools 3.0

§       eMbedded Visual C++ 4.0

§       Visual Studio .NET 2003

§       The SDK also requires that you close the emulators provided with eMbedded Visual Tools, eMbedded Visual C++, or Visual Studio .NET. Under some circumstances the Windows CE Emulator, or related processes may continue to run even though it seems the applications have exited. It may be advisable to reboot your computer before proceeding with the installation to ensure no conflicts with existing running programs occur.

2. Uninstall Previous Versions of the Smartphone 2003 SDK

You do not need to uninstall any version of the Smartphone 2002 SDK to use the Smartphone 2003 SDK, but any previous versions of the Smartphone 2003 SDK do need to be uninstalled. If you are upgrading from a previous version of the Smartphone 2003 SDK, after uninstalling the previous version but before installing the current version, run eMbedded Visual C++ 4.0 and view the Help contents. After this is done, close Help and the eMbedded Visual C++ 4.0 environment and install the new Smartphone 2003 SDK. This ensures that the latest Help documentation gets installed into the eMbedded Visual C++ 4.0 environment.

3. Install Microsoft ActiveSync 3.7

The Smartphone 2003 SDK requires ActiveSync 3.7 for application development and deployment using eMbedded Visual C++ 4.0 or Visual Studio .NET 2003.  You must first uninstall previous versions of ActiveSync before installing ActiveSync 3.7. If you do not have ActiveSync 3.7, you may obtain it from http://www.microsoft.com/mobile/pocketpc/downloads.

If you already have Visual Studio .NET 2003 installed, and then uninstall and reinstall ActiveSync 3.7, you will have to Repair your installation of Visual Studio .NET 2003 from Add/Remove Programs.

4. Install eMbedded Visual C++ 4.0 and Service Pack 2

In order to develop C++ applications using the Smartphone 2003 SDK, you must install Microsoft eMbedded Visual C++ 4.0 and Service Pack 2 before installing the Smartphone 2003 SDK. If you do not already have eMbedded Visual C++ 4.0 or Service Pack 2 you may download them from http://msdn.microsoft.com/vstudio/device/embedded/download.asp.

Note, installing eMbedded Visual C++ 4.0 Service Pack 2 on Windows Server 2003 causes a pop-dialog box to come up and prompt that one of the emulator drivers isn't signed. You should accept the unsigned driver and continue.

5. Install Visual Studio .NET 2003

In order to develop applications in C# or VB.NET for Smartphone 2003, you must install Visual Studio .NET 2003 before installing the SDK.

Smartphone 2003 devices and the Smartphone 2003 Emulation images already contain the .NET Compact Framework.

For more information on Visual Studio .NET 2003 or the .NET Compact Framework, see: http://msdn.microsoft.com/vstudio/.

6. Install the Smartphone 2003 SDK

You can now proceed with the installation of the Smartphone 2003 SDK.

For updates, tools, and whitepapers please check the Microsoft Mobile Device Developers website: http://www.microsoft.com/mobile/developer.

 

 

eMbedded Visual C++ 4.0

Getting Started

This section walks you through the creation of a simple Hello World application in eMbedded Visual C++ 4.0 using the emulator.

1.     Open eMbedded Visual C++ 4.0.

2.     From the File menu, choose New.

3.     In the New dialog, select WCE Smartphone 2003 Application in the Projects tab.

4.     In the Project Name text box, enter TestSDK as the name for your application.

5.     Ensure the Win32 (WCE emulator) check box in the bottom right corner of the New dialog is selected. (Including the x86 target in the project allows you to build the application for the emulator).

6.     In the first step of the application wizard, select A typical 'Hello World' application and click Finish.

7.     In the New Project Information dialog, click OK.

8.     For the active WCE Configuration choose Smartphone 2003 and Win32 (WCE emulator) Release. For the Device drop-down choose Smartphone 2003 Emulator to send the application to the emulator.

9.     Select Build-Rebuild All.

 

Targeting your Smartphone Device

Before using eMbedded Visual C++ 4.0 with your smartphone device you’ll need to run the Smartphone Development Package Signer tool (spdps.exe) found in the SDK tools folder.  For most users the command will be “spdps /device /create”.  Please refer to the “Using the Development File Signing Tool” document for further information on using the Smartphone Development Package Signer tool.

Windows CE Emulator and Smartphone 2003 Emulator Images

Emulator Image Failure (“Cannot obtain DHCP address”) when Booting an Image using Virtual Switch Networking Support

When using the Virtual Switch networking option with the emulator, on some occasions the emulator image may fail to boot correctly (as a result of not being able to obtain an IP Address from a DHCP Server). To work around this issue, shut down the emulator (and ensure the emulator process pbemulator.exe is no longer running in Task Manager), and restart the emulator.

Using the Radio Add-On Pack

A radio add-on pack can be purchased directly from http://www.wavecom.com/.  When installed, the radio will enable you to make phone calls, send and receive SMS messages, and so on, from the emulator.  For more details, use the SDK documentation index to find the topics “Wireless GSM Telephony in the Emulator” and “Switching the Emulator Image to a Radio or Non-Radio Build.”

Configuring the Emulator to Access the Internet

For more information about configuring the emulator to access the Internet, configuration related to proxy servers, and MSN Instant Messenger, see the “Configuring the Smartphone 2003 Connection Manager” section in the SDK documentation.

Using ActiveSync with the Emulator

For information about using ActiveSync with the emulator see the “Using ActiveSync with the Emulator” section of the SDK documentation.

Starting the Emulator from the Command Line

For information about starting the emulator from the command line see the “Emulator Command Line Parameters” section of the SDK documentation.

Emulator Clock and WM_TIMER Messages

Under some circumstances the emulator clock may be running slightly slower or faster than the clock on the host PC.  This can result in different date/time being reported in the emulator than is report on the host PC.

In addition, the emulator timer (SetTimer) that generates WM_TIMER messages does not work the same as a real device. As a result, WM_TIMER messages may not be generated as often. This can cause some difficulty when working with timer dependent applications in the emulator.

English-Only Characters in Edit Control

Expect to see English-only characters when entering characters into an edit control in an eEmulator image, even if your desktop locale or keyboard configuration is not English. The keyboard driver for the emulator is provided only in English.

Visual Studio .NET 2003 Smartphone Support

Prerequisites

ActiveSync 3.7 or greater and Visual Studio .NET 2003 must be installed prior to installing the Smartphone 2003 SDK to enable Smartphone device support in Visual Studio .NET.

Upgrading ActiveSync

If you are upgrading ActiveSync from a previous version, and Visual Studio .NET is installed, it is important that you NOT uninstall the previous version.  Install ActiveSync as an upgrade.  If you uninstall ActiveSync it will cause errors in the installation of Visual Studio .NET.  If this happens you can remove, then re-install Smart Device Project support in maintenance mode in Visual Studio .NET Setup.

Creating a Smartphone Project

After installing the SDK, the Smartphone platform becomes an additional option within Smart Device Application projects for C# and VB .NET.  To create a Smart Device Application, choose File menu, then click New Project, then choose Visual Basic Projects or Visual C# Projects, then choose Smart Device Application.  You can then use the Application Wizard to choose to create an application based on the Smartphone platform.

Accessing Help

Help topics specific to development for the .NET Compact Framework is available within the Visual Studio .NET help collection.  See Smartphone 2003 Help in the Visual Studio Help Contents.  General development help is available from the Start menu.  Choose Start, All Programs, Microsoft Smartphone 2003 SDK, Smartphone 2003 Help.

DataGrid Control not supported on Smartphone

The DataGrid control is enabled as an option in the Device Toolbox in a Smartphone project; however, this control is not supported on Smartphone 2003, and will cause an error at build time if it is used.

Modifying the tab order on a Smartphone form.

The Smartphone forms designer requires the Smartphone Tab Order tool to adjust the tab order.  Choose View, Smartphone Tab Order to access this tool.  The other Tab Order tool on the View menu does not work with a Smartphone project.

Building a distributable CAB file for Smartphone

CAB files for Smartphone must be built using the CabWizSP.exe command line tool.  The Build CAB tool in Visual Studio .NET is not compatible with Smartphone.  This file is found in the Tools directory of the SDK install location.  By default, this is in \Program Files\Windows CE Tools\wce420\SMARTPHONE 2003\Tools.  See the Smartphone 2003 Help documentation for details.

Some unsupported control properties or events appear in the Property Browser

The property browser displays some properties that are not supported on the Smartphone.  The ContextMenu property is not supported on any control or form.  The MinimizeBox, MaximizeBox, and ToolBar properties are not supported on a form.  The CheckBoxes property is not supported on the TreeView control.

Some controls will appear differently at runtime than at design time.

The forms designer displays several controls using a standard Desktop application appearance; however their appearance at runtime on the Smartphone or emulator will conform to the Smartphone UI guidelines.  These include the MainMenu, ScrollBar, ComboBox, and ProgressBar controls, as well as the form itself.  If the WinForms designer Font property is set to a font that is not supported on the device or emulator, the default font for the device will be rendered at runtime.

Changing Smartphone emulator configuration results in screen height error.

Making a change to the emulator configuration results in an error dialog that states, “Screen height value must be any multiple of 8 between 64 and 768.”   To work around this issue, set the values to a multiple of eight.  Note that these values are over-ridden by values in the emulator skin files, and have no effect.

Some unsupported properties, methods, or events may appear in Intellisense help or the VS shell.

In some cases, the Intellisense help, the property browser, or the VB.NET control events drop-down list may display properties, methods, or events that are not supported by the Smartphone or by the .NET Compact Framework.

 

Attempting to hook an unsupported event will result in the event handler not firing at runtime.

 

For a list of properties, methods, and events supported by the .NET Compact Frameworks, please consult the Smart Devices help in the MSDN Online help in Visual Studio .NET.  For Smartphone-specific questions, see the Smartphone SDK documentation.

Deploying Satellite dlls to a locked device or emulator is not supported

During the deployment process, Visual Studio .NET signs the project output assemblies, but does not sign the resource satellite dlls with a certificate.  If the device or emulator is locked and the device trusts the certificate, the satellite dlls will fail to load at runtime.  To work around this, manually sign the project files using SignCode.exe and create an Application Deployment CAB file using CABWizSP.exe. See the section Security in Managed Code in the Visual Studio .NET, Smartphone 2003 Help.

 

Using the Smartphone 2003 device

Before Visual Studio .NET 2003 can deploy and debug managed applications on your Smartphone device you’ll need to do one of the following:

  • Provision your Smartphone device with the SDKTestCerts.xml document contained in the SDK tools folder.  For more information on provisioning a device refer to the SDK documentation.
  • Update the Visual Studio .NET 2003 configuration file to use a certificate already trusted on the device.  Open the file “microsoft.smartphone2003.1.0.xsl” located at %ALLUSERSPROFILE%\Application Data\Microsoft\visualstudio\devices\addons.  Find the “SignArgs” property and replace it with all appropriate command line options for signcode.exe.  For more information on signcode.exe visit http://msdn.microsoft.com/library/default.asp?url=/workshop/security/authcode/signing.asp.

 

Using the Smartphone 2003 Emulator (Radio Required) emulator image

1.   Connect the radio to power and a COM port on the desktop machine.

2.   Start Visual Studio .NET 2003.

3.   From the Tools menu chose Options…

4.   Select the Device Tools item, Devices sub item.

5.   Change the Show devices for platform: to Smartphone.

6.   Select the Smartphone 2003 Emulator item.

7.   Click the Configure…button to the right of the Startup server: listbox control.

8.   Click the Hardware tab.

9.   Set Serial port 2: to the COM port the radio was connected to in step 1.

10. Click the OK button.

11. Click the OK button.

 

Note: If you have used the Smartphone 2003 Emulator image before you will need to close the emulator without saving state for the changes to take effect.

Setting the correct font and font size for double byte character languages

The default font for Smartphone projects is Nina 11pt, which is not included on devices or emulators whose localization language uses a double-byte code page. In this case, the device will render characters using the default font for the device and the default font size for the Smartphone project, which is 11pt. To be able accurately set fonts at run-time, set the Font property in the Properties window to a font supported by the emulator.

TextBox and TreeView controls can’t display double-byte characters in the WinForms designer for the Nina Font

To display readable double-byte characters at design-time in the TextBox and TreeView controls, set the Font property to a font supported both on the device and desktop.  Note that this is a design-time only issue – runtime behavior is not affected.

Correcting the Help filter in non-English versions of Visual Studio .NET

Smartphone Help topics do not appear in non-English versions of Visual Studio .NET when .NET Compact Framework is selected in the Filtered by box. To correct this problem, edit the .NET Compact Framework filter:

1.   On the Help menu, click Edit Filters.

2.   In the Filter box, select .Net Compact Framework.

3.   In the Filter Definition box, remove the "Locale" clause. For example, in the Japanese version of Visual Studio .NET, you would remove "AND 'Locale'='KBJapanese'".

Note   The language specified in the "Locale" clause depends on the language of Visual Studio .NET.

4.   Click Save.
